# Architecture

## Overview

`sample-terminal` is a native macOS terminal prototype written in Rust. It uses:

- `objc2` bindings to drive AppKit for windowing and event handling.
- a PTY-backed shell session for terminal I/O.
- an in-process terminal model for ANSI parsing, scrollback, cursor state, and resize behavior.
- Metal for terminal rendering, including custom geometry generation and a glyph atlas.

At a high level, the runtime flow is:

1. [`src/main.rs`](./src/main.rs) calls `app::run()`.
2. [`src/app.rs`](./src/app.rs) creates the window, view, timer, and global `AppState`.
3. [`src/app_state.rs`](./src/app_state.rs) owns the `TerminalSession`, `TerminalBuffer`, blink state, and selection state.
4. [`src/session.rs`](./src/session.rs) spawns the shell in a PTY and exposes nonblocking reads plus input/resize writes.
5. [`src/terminal_buffer/mod.rs`](./src/terminal_buffer/mod.rs) and its submodules parse PTY bytes into terminal state and produce a `RenderSnapshot`.
6. [`src/renderer/metal.rs`](./src/renderer/metal.rs), [`src/renderer/cells.rs`](./src/renderer/cells.rs), and [`src/renderer/atlas.rs`](./src/renderer/atlas.rs) convert the snapshot into Metal draw calls.

## Module Relationships

- `app` is the AppKit bridge and render loop.
- `app_state` is the central runtime state coordinator.
- `session` talks to the shell process through a PTY.
- `terminal_buffer` is the terminal emulator core.
- `layout` translates between window coordinates and grid coordinates.
- `renderer` turns terminal state into GPU-ready geometry and submits frames.
- `input` translates AppKit events into terminal input and selection state updates.

## Runtime Flow

### Startup

- `main()` delegates to `app::run()`.
- `run()` creates an `AppState` sized from the initial window dimensions.
- `TerminalView::finish_setup()` creates a `TerminalRenderer`, installs its `CAMetalLayer`, renders once, and updates the window title with the Metal device name.

### Input and PTY Traffic

- Keyboard events flow through `key_down_impl()`, then `translate_terminal_input()`, then `AppState::send_input()`, then `TerminalSession::write_input()`.
- PTY output is collected by the session reader thread and drained by `AppState::poll_session()`.
- `TerminalBuffer::push_bytes()` feeds the parser one byte at a time and mutates the active screen.

### Rendering

- A 60 Hz `NSTimer` calls `tick_impl()`.
- `tick_impl()` polls PTY output and blink state through `AppState::poll_session_and_should_render()`.
- `render_view()` computes grid dimensions, syncs PTY window size, asks `AppState` for a `RenderSnapshot`, and calls `TerminalRenderer::render()`.
- `TerminalRenderer` updates caches, builds draw batches, and encodes three passes:
  - solid background/chrome
  - text glyph instances
  - overlays such as selection and cursor

### `src/main.rs`

Purpose:
- Thin process entry point.

Important methods:
- `main()`: calls `app::run()`, prints any startup error, and exits with status `1` on failure.

### `src/app.rs`

Purpose:
- Owns AppKit application startup, the custom `NSView` subclass, event handling, and render scheduling.

Important structs:
- `TerminalViewState`: per-view state, currently holding the `TerminalRenderer` and any startup error.
- `TerminalView`: custom `NSView` subclass defined with `define_class!`; receives AppKit events and delegates into Rust logic.
- `TimerTarget`: `NSObject` target for the repeating `NSTimer`.

Important methods and functions:
- `run()`: top-level application bootstrap; initializes global `APP_STATE`, creates the window/view, installs the timer, and runs the AppKit loop.
- `create_window()`: configures the `NSWindow`.
- `TerminalView::new()`: allocates the custom view and runs setup.
- `TerminalView::finish_setup()`: creates `TerminalRenderer`, installs the Metal layer, triggers the first render, and sets the window title.
- `key_down_impl()`: converts AppKit key events into terminal byte sequences.
- `selection_cell()`: maps a mouse location to a terminal cell.
- `selection_event()`: updates selection state and requests a render.
- `scroll_event()`: converts wheel deltas into viewport scrolling.
- `render_view()`: main render coordinator; derives geometry, syncs PTY size, requests a `RenderSnapshot`, and submits a frame.
- `tick_impl()`: timer callback that polls PTY output and triggers redraws when activity or blink state changed.

Important AppKit overrides on `TerminalView`:
- `isFlipped()`: keeps view coordinates top-left oriented.
- `acceptsFirstResponder()`: enables keyboard focus.
- `wantsUpdateLayer()` / `updateLayer()`: layer-backed drawing path.
- `renderFrame()`: custom Objective-C selector used by the timer path.
- `keyDown:`, `mouseDown:`, `mouseDragged:`, `mouseUp:`, `scrollWheel:`: event entry points.
- `viewDidChangeBackingProperties()` and `setFrameSize:`: trigger rerendering on DPI or size changes.

### `src/app_state.rs`

Purpose:
- Central shared application state between the AppKit layer, PTY session, and terminal model.

Important structs:
- `AppState`: owns the `TerminalSession`, `TerminalBuffer`, window-size cache, activity counter, cursor blink state, and selection state.
- `BlinkState`: tracks cursor blink visibility and reset-on-activity behavior.
- `SelectionState`: stores selection anchor/focus and drag state.

Important methods:
- `AppState::new_for_window()`: computes initial grid size from window dimensions.
- `AppState::new()`: creates session, terminal buffer, and state holders.
- `poll_session()`: drains PTY output and pushes it into the `TerminalBuffer`.
- `poll_session_and_should_render()`: combines PTY activity and blink timing to decide whether a render is needed.
- `send_input()`: resets viewport/selection, bumps activity, and writes bytes to the PTY.
- `sync_window_size()`: avoids redundant PTY resize ioctls by caching the last sent size.
- `cursor_visible()`: applies blink logic to the cursor.
- `selection_range()`: converts internal selection state into renderer-facing `SelectionRange`.
- `update_selection()`: applies `SelectionPhase` state transitions.
- `scroll_viewport()`: changes the scrollback viewport offset.
- `stop_selection_drag()`: terminates drag tracking after scroll.
- `render_snapshot()`: resizes the terminal buffer if needed and extracts a renderer-facing snapshot.
- `BlinkState::sync_activity()`: forces the cursor visible after input/output activity.
- `BlinkState::tick()`: toggles blink visibility after the configured interval.
- `SelectionState::range()`: normalizes anchor/focus presence into an optional selection.

### `src/input.rs`

Purpose:
- Input translation helpers shared by the AppKit event layer.

Important items:
- `SelectionPhase`: selection state machine inputs (`Start`, `Update`, `End`).
- `translate_terminal_input()`: converts user text and AppKit arrow-key constants into terminal byte sequences.
- `reduce_selection_phase()`: pure reducer for selection anchor/focus/dragging updates.
- `normalize_scroll_lines()`: converts AppKit scroll deltas into line-based viewport movement.

### `src/layout.rs`

Purpose:
- Terminal layout constants and coordinate math shared by input handling and rendering.

Important structs:
- `LayoutMetrics`: computed terminal chrome/content geometry for a given view size and grid size.

Important methods:
- `terminal_grid_size()`: computes terminal columns and rows from the view size and fixed padding/cell metrics.
- `layout_metrics()`: computes pixel-space rectangles for the outer frame, header, and content grid.
- `point_to_cell()`: maps a view-space point into a `(row, col)` when the point is inside the grid.

### `src/session.rs`

Purpose:
- PTY session management and shell process lifecycle.

Important structs:
- `TerminalSession`: owns the PTY master file descriptor and a receiver for asynchronously collected PTY output.

Important methods:
- `TerminalSession::spawn()`: creates the PTY with `forkpty`, execs the login shell in the child, and starts reader/reaper threads in the parent.
- `TerminalSession::try_read()`: drains any pending PTY output chunks without blocking.
- `TerminalSession::write_input()`: writes bytes to the PTY master.
- `TerminalSession::resize()`: issues `TIOCSWINSZ` to update the shell’s terminal size.
- `default_shell_path()`: prefers `$SHELL`, then falls back to `/bin/zsh`.
- `spawn_reader_loop()`: background thread that repeatedly reads PTY output and sends chunks over a channel.
- `spawn_reaper()`: waits for child process termination to avoid leaving a zombie.

### `src/renderer/mod.rs`

Purpose:
- Renderer module root and public re-export surface.

Important items:
- Re-exports `cells` types used by the terminal model: `Cell`, `CursorState`, `ActiveScreen`, `RenderDamage`, `RenderSnapshot`, `SelectionRange`.
- Re-exports `metal` entry points: `RenderFrameInput` and `TerminalRenderer`.

### `src/renderer/cells.rs`

Purpose:
- Renderer-side data model and the CPU geometry builder for terminal chrome, rows, selections, and cursor.

Important structs and enums:
- `Color`: small wrapper around RGBA float arrays used internally for quads.
- `Cell`: renderer-facing terminal cell contents.
- `CursorState`: cursor row/column/visibility snapshot.
- `ActiveScreen`: whether the terminal is showing the primary or alternate screen.
- `RenderDamage`: row/global/full-rebuild dirty state from the terminal model.
- `RenderSnapshot`: immutable renderer input containing dimensions, cells, cursor, active screen, and damage info.
- `SelectionRange`: inclusive rectangular/text-style selection endpoints with normalization helpers.
- `Quad`: solid-color rectangle primitive.
- `TextInstance`: per-glyph instance data for the text draw pass.
- `RowGeometry`: cached geometry for a single terminal row.

Important methods:
- `RenderSnapshot::new()`: allocates a blank snapshot and marks it as requiring a full rebuild.
- `RenderSnapshot::set_cell()`, `set_char()`, `set_cursor()`, `set_active_screen()`: snapshot mutation helpers used by tests and the terminal model.
- `RenderSnapshot::cell()`, `char_at()`: read helpers used by tests and render code.
- `SelectionRange::normalized()`: orders endpoints from start to end.
- `SelectionRange::contains()`: tests whether a given cell lies in the selection.
- `build_chrome_quads()`: returns the outer frame, border, header, and grid background quads.
- `build_row_geometry()`: turns one terminal row into background tiles, glyph instances, and selection overlays.
- `build_cursor_quad()`: emits the cursor overlay quad when the cursor should be drawn.

### `src/renderer/atlas.rs`

Purpose:
- Font loading, glyph rasterization, atlas packing, and Metal texture construction.

Important structs:
- `FontMetrics`: ascent/descent/baseline/inset data derived from CoreText.
- `GlyphMetadata`: atlas UVs, bitmap dimensions, offsets, and advance for a glyph.
- `GlyphAtlas`: owns the Metal glyph texture, the glyph metadata map, fallback glyph metadata, and font metrics.

Important internal structs:
- `RasterizedGlyph`: bitmap plus typographic metrics for one character before packing.
- `AtlasPlacement`: packed texture origin for a glyph.
- `AtlasPack`: packed atlas bytes, size, and per-glyph placements.

Important methods:
- `GlyphAtlas::new()`: full atlas build pipeline; loads the font, rasterizes supported glyphs, packs them, uploads the texture, and records metadata.
- `GlyphAtlas::texture()`: exposes the Metal texture.
- `GlyphAtlas::font_metrics()`: exposes computed font metrics.
- `GlyphAtlas::glyph_for()`: resolves metadata for a character with fallback behavior.
- `load_font()`: loads the Menlo font via CoreText.
- `compute_font_metrics()`: derives atlas layout metrics from the font.
- `build_glyph_rasters()`: rasterizes all supported characters.
- `glyph_for_char()`, `glyph_metrics()`, `glyph_advance()`: CoreText metric lookup helpers.
- `bitmap_dimensions()`: derives raster dimensions from glyph bounds.
- `rasterize_glyph()`: draws the glyph into a grayscale bitmap with Core Graphics.
- `pack_glyphs()`: lays out glyph bitmaps into a 2D atlas with row packing.
- `build_texture()`: uploads atlas bytes into a Metal `R8Unorm` texture.
- `supported_chars()`: currently returns printable ASCII plus Latin-1 supplement characters.

### `src/renderer/metal.rs`

Purpose:
- Metal device setup, shader compilation, cache management, frame resource allocation, and draw submission.

Important structs:
- `RenderFrameInput`: view-space/pixel-space frame parameters, scale factor, cursor visibility, and selection state.
- `TerminalRenderer`: the main renderer; owns the Metal device objects, glyph atlas, drawable size, cached chrome, cached row geometry, and last selection.

Important internal structs:
- `SolidVertex`: vertex format for solid quad drawing.
- `TextUniforms`: per-frame text shader uniforms containing the view size.
- `FrameResources`: current drawable, command buffer, and encoder.
- `DrawBatches`: collected background quads, text instances, and overlay quads for one frame.

Important methods:
- `TerminalRenderer::new()`: creates the Metal device, command queue, shader library, pipeline states, sampler, atlas, and `CAMetalLayer`.
- `TerminalRenderer::layer()`: exposes the backing `CAMetalLayer` for installation into the view.
- `TerminalRenderer::resize()`: updates layer frame, content scale, and drawable size.
- `TerminalRenderer::render()`: frame entry point; updates drawable state, refreshes caches, collects draw batches, encodes draws, and presents.
- `TerminalRenderer::device_name()`: returns the Metal device name for display in the window title.
- `update_caches()`: maintains chrome and per-row geometry caches based on `RenderDamage` and selection changes.
- `update_drawable()`: thin wrapper around `resize()`.
- `prepare_frame_resources()`: gets the next drawable, configures the render pass, and creates the command encoder.
- `update_layout_caches()`: derives `LayoutMetrics` and updates cached geometry.
- `collect_draw_batches()`: flattens cached row geometry into per-frame arrays and appends the cursor overlay.
- `encode_draw_batches()`: runs the solid-text-solid pass sequence.
- `present_frame()`: ends encoding, presents the drawable, and commits the command buffer.
- `build_solid_pipeline()`, `build_text_pipeline()`, `build_text_sampler()`: create the Metal pipeline state objects.
- `draw_solid_pass()`, `draw_text_pass()`: per-pass encoding helpers.
- `make_buffer()`: uploads arbitrary data into a Metal buffer.
- `build_scene_vertices()`, `push_rect()`, `to_ndc_x()`, `to_ndc_y()`: CPU-side quad tessellation and coordinate conversion helpers.

### `src/terminal_buffer/mod.rs`

Purpose:
- Root terminal emulator state object and renderer snapshot extraction.

Important structs:
- `TerminalBuffer`: owns both screens, scrollback, parser state, UTF-8 partial buffer, current attributes, viewport offset, and damage tracking.

Important methods:
- `TerminalBuffer::new()`: initializes the primary/alternate buffers and marks the whole screen dirty.
- `resize()`: resizes both screens, resizes scrollback rows, clamps viewport state, and marks a full rebuild.
- `cols()`, `rows()`: expose current active-screen dimensions.
- `push_bytes()`: feeds PTY bytes into the parser.
- `set_viewport_offset()`, `viewport_offset()`: manage scrollback viewport position.
- `scrollback_len()`: exposes scrollback depth.
- `render_snapshot()`: builds a renderer-facing `RenderSnapshot`, including visible rows and cursor visibility rules.
- `screen()`, `screen_mut()`: choose the active primary or alternate screen.

### `src/terminal_buffer/types.rs`

Purpose:
- Core terminal data types shared across parser, ops, and color modules.

Important structs and enums:
- `TerminalCell`: terminal-model cell contents, including colors and flags.
- `CellAttributes`: current SGR attributes used for future writes.
- `ParserState`: parser state machine modes (`Ground`, `Escape`, `Csi`, `Osc`, `OscEsc`).
- `ScreenBuffer`: one concrete screen grid plus cursor, wrap, saved cursor, and scroll region state.
- `ParsedCsi<'a>`: parsed CSI parameters passed into dispatch.

Important methods:
- `TerminalCell::default()`: blank cell with default foreground/background colors.
- `From<TerminalCell> for renderer::Cell`: converts terminal cells into renderer cells.
- `ScreenBuffer::new()`: creates a blank screen and initializes scroll region defaults.
- `ScreenBuffer::resize()`: resizes the grid while preserving the top-left overlap.
- `ScreenBuffer::clear()`: clears the screen and resets cursor/scroll-region state.
- `ScreenBuffer::index()`, `cell()`, `set_cell()`, `clear_row()`: low-level cell access helpers.

### `src/terminal_buffer/damage.rs`

Purpose:
- Internal dirty-state tracking between terminal mutations and renderer cache updates.

Important structs:
- `DamageTracker`: tracks full rebuilds, dirty rows, cursor dirtiness, selection dirtiness, and global dirtiness.

Important methods:
- `mark_full_rebuild()`: records that renderer caches must be fully rebuilt.
- `mark_row()`, `mark_rows()`, `mark_all_rows()`: mark row-level damage.
- `mark_cursor_dirty()`: flags cursor-related redraw work.
- `take()`: converts internal state into renderer-facing `RenderDamage` and resets the tracker.

Note:
- `selection_dirty` exists in the tracker shape but is not actively marked by the current code.

### `src/terminal_buffer/color.rs`

Purpose:
- ANSI color decoding and SGR attribute updates.

Important methods:
- `ansi_16_color()`: returns themed RGBA values for the 16 basic ANSI colors.
- `parse_extended_color()`: parses `38`/`48` extended color payloads for 256-color and truecolor sequences.
- `ansi_256_color()`: maps 256-color indices into RGBA values.
- `TerminalBuffer::set_graphics_rendition()`: applies CSI `m` parameters to `current_attr`.

### `src/terminal_buffer/parser.rs`

Purpose:
- Byte-wise terminal parser and CSI/OSC dispatch.

Important methods:
- `TerminalBuffer::push_byte()`: top-level parser state machine entry point.
- `handle_ground_byte()`: handles printable bytes and control characters in normal parsing mode.
- `handle_escape_byte()`: handles `ESC` sequences and transitions into CSI/OSC states.
- `handle_csi_byte()`: accumulates CSI bytes until the final byte is reached.
- `handle_osc_byte()`, `handle_osc_esc_byte()`: consume OSC payloads until termination.
- `execute_csi()`: dispatches supported CSI sequences into terminal operations.
- `push_utf8_byte()`: incrementally decodes multi-byte UTF-8 sequences.
- `flush_utf8_lossy()`: emits a replacement character for incomplete pending UTF-8 data.
- `parse_csi()`: parses raw CSI parameter bytes into `private` mode and optional integer params.
- `param_or()`: parameter helper with defaults.

Supported behavior currently includes:
- cursor movement and direct positioning
- erase in line / erase in display
- insert/delete characters
- insert/delete lines
- SGR color handling
- scroll-region changes
- save/restore cursor
- DEC private mode toggles for alternate screen and cursor save/restore

### `src/terminal_buffer/ops.rs`

Purpose:
- Terminal mutation operations used by the parser.

Important methods:
- `reset()`: clears both screens, parser state, attributes, scrollback, and resets to the primary screen.
- `write_char()`: writes a character at the cursor using current attributes, including wrap handling.
- `carriage_return()`, `line_feed()`, `next_line()`, `backspace()`, `tab()`: basic control-character operations.
- `index_down()`, `reverse_index()`: scroll-aware vertical movement.
- `move_cursor_relative()`, `set_cursor_position()`, `set_cursor_row()`, `set_cursor_col()`: cursor positioning helpers.
- `erase_in_display()`, `erase_in_line()`: clearing operations.
- `insert_chars()`, `delete_chars()`: shift cells horizontally.
- `insert_lines()`, `delete_lines()`: shift rows within the active scroll region.
- `set_scroll_region()`: updates the active scrolling bounds.
- `save_cursor()`, `restore_cursor()`: save and restore cursor position.
- `scroll_up()`, `scroll_down()`: region-aware scrolling wrappers.
- `scroll_up_in_region()`: scrolls upward, pushing rows into scrollback when operating on the full primary screen.
- `scroll_down_in_region()`: scrolls downward inside the current region.
- `clear_row()`, `set_blank()`: mutation helpers that also mark damage.
- `set_private_mode()`: dispatches DEC private modes such as alternate-screen entry/exit and cursor save/restore.
- `visible_rows()`: builds the current viewport by combining scrollback with the active live screen.
- `resize_scrollback_rows()`: resizes stored scrollback rows after terminal width changes.

## Testing Layout

Tests are colocated inside the implementation files. The current coverage is concentrated in:

- `src/input.rs`
- `src/layout.rs`
- `src/app_state.rs`
- `src/renderer/cells.rs`
- `src/renderer/atlas.rs`
- `src/terminal_buffer/mod.rs`

These tests mainly exercise parser behavior, resize behavior, geometry generation, blink logic, color handling, and glyph atlas construction.

## Architectural Notes

- The codebase is split cleanly by responsibility: AppKit/UI glue, shared state, PTY/session I/O, terminal emulation, layout, and rendering.
- `AppState` is the coordination boundary between the UI thread and the terminal/session state.
- `TerminalBuffer` is effectively the emulator core; parser, ops, color, damage, and types are deliberately factored into submodules but mutate the same root struct.
- Rendering is snapshot-based: the terminal model emits `RenderSnapshot`, and the renderer keeps its own geometry caches keyed by damage state.
- Scrollback only applies to the primary screen; the alternate screen renders directly from its own buffer.
